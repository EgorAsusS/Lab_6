# Lab_6

Реализация ПОЛИЗ  для алгебраического выражения

ПОЛИЗ - запись, которая обладает такими свойствами:
1. бесскобочная запись
2. операнды – в том же порядке, что и в исходном выражении
3. операции – в порядке их выполнения
4. операции располагаются сразу после своих операндов – постфиксная
запись (инверсная)

Приоритеты операций
● ( – приоритет 0
● ) – приоритет 1
● = – приоритет 2 (стековый), 9 (сравнительный)
● + – – приоритет 6
● * / – приоритет 7

int check_el(char el);
Функция принимает на вход символ и возвращает тип этого элемента в соответствии с таблицей
Функция возвращает значение типа int - тип этого символа
Таблица типов

Код | Тип символа
0   | Неправильный символ
1   | Переменная (a-z, A-Z)
2   | Константа (0-9)
3   | Операция (+, -, *, /, =)
4   | Скобки - ()


char* form_POLIS(char* input);
Функция получает на вход строку, по которой будет строиться ПОЛИЗ и возвращает указатель на строку, в которую формируется ПОЛИЗ

Функция работает по алгоритму Дейкстры (построения ПОЛИЗ):

Исходное выражение просматривается последовательно, как строка
Операнды сразу заносятся в ПОЛИЗ, а операции - через стек, по следующим
правилам
1. если стек пуст, то любой знак сразу заносится в стек
2. знаки с нулевым приоритетом заносятся сразу в стек
3. для знаков с ненулевым приоритетом выполняется сравнение входного
знака и знака в вершине стека
если приоритет входного знака больше приоритета знака в вершине стека,
то входной знак заносится в стек
иначе входной знак сначала выталкивает из стека в ПОЛИЗ все знаки с
бóльшим или равным приоритетом, а затем заносится в стек
4. скобка ‘(’, сразу заносится в стек; скобка ‘)’ выталкивает из стека все знаки до
ближайшей ‘(‘ по общему правилу; далее скобки взаимно уничтожаются
5. когда закончится выражение, стек очищается: все знаки из него
выталкиваются в ПОЛИЗ

Работа с унарными плюс и минус ведется так: унарный плюс пропускается, а для унарного минуса происходит преобразование до бинарной операции, ставится 0 перед минус
(-a) -> (0-a). Также недопустимым считается два любых знака операции подряд
Функция возвращает указатель на строку, содержащую ПОЛИЗ, либо NULL в случае ошибки

Ошибки, при которых будет возвращаеться NULL:
На вход пришел NULL указатель на исходную строку
Выход за размеры буфера
Русские символы
Ошибка функций работы со стеком




int calculate_POLIS(char* str, int* Tab, double* answer);
Функция получает на вход строку, по которой будет вычисляться ПОЛИЗ, ассоциативный массив Tab, где для переменных заданы значения, указатель на переменную answer,
в которую заноситься окончательный результат.

Вид аргументов
Ассоциативный массив int* Tab[256]
Строка char* str типа Си, в конце строки '\0'
Если аргументы не такого вида, то поведение функции неопределено

Алгоритм:
1. Строим ПОЛИЗ как строку
2. Операнды заносим в стек
3. При встрече знака операции извлекаем из стека нужное число операндов,
применяем к ним операцию и результат заносим в стек
● Операнды извлекаются в обратном порядке. Так как работа через стек

Если при построении ПОЛИЗ было передано уравнение (т.е. например y = a + b), то в Tab[0] удет храниться код символа, которому присвоили значение (для данного примера 121).

Функция возвращает значение ошибки типа int
Коды возвращаемых значений

Код | Описание ошибки
1   | Успешное вычисление
0   | Ошибка функций работы со стеком, т.е. невозможно посчитать ПОЛИЗ
-1  | На входе функции нет аргументов
-2  | Ошибка операций (деление на 0 и const = variable)
-3  | Неправильные символы в исходной строке



char* read_file(char* filepath, int** Tab);

Функция чиатет файл по пути filepath, заполняет ассоциативный массив Tab значениями переменных и формирует строку с выражением.
Сам массив Tab создается и заполняется внутри функции, так как на вход передается двойной указатель, который хранить адрес указателя,
через который будет вестись дальнейшая работа с Tab. Формат файла выражение, которое нужно посчитать, либо имя переменной = выражение.
Внутри функции также динамически запрашивается память для буфера, в который будет заполняться считанное выражение из файла. Размер буфера 2000 символов
Функция возвращает указатель на строку, содержащую выражение, либо NULL в случае ошибки

Ошибки, при которых будет возвращаеться NULL:
Файл не открывается, на вход пришел int** Tab == NULL
Не получилось считать символы из файла
Выход за размеры буфера
Русские символы, некорректный формат
Ошибка fclose
